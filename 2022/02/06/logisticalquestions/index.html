CF566C Logistical Questions

Posted on 2022-02-06 | Post modified: 2022-07-28 | In Graph Theory
Words count in article: 748 character(s) | Reading time ≈ 4 minute(s)

https://codeforces.com/problemset/problem/566/C

We are given a unrooted tree consists of $n(n\leq200\ 000)$ vertices labelled from $1$ to $n$. The $i$ -th vertex has a non-negative weight $w_i(0\leq w_i\leq10^8)$. Each edge has a length $l_i(1\leq l_i\leq1000)$. Let $Dist(u,v)$ be the sum of all edge’s length on path from $u$ to $v$. We need to select a root $c$ in order to minimize

First, consider randomly select a root $u$ which has $k$ children $v_1,\cdots,v_k$. Either $u$ is the root $c$ we desire or $c$ is in one of its child $v_i$. Each time we need to come up with an idea to specify this particular $v_i$, so the possible root $c$ will be limited within nodes in subtree $v_i$ (and node $u$).

Second, we need to realize that $ans$ for root $u$ consists of $n$ parts, each of them resembles

For any path $a\to b$ in the tree and $x\in a\to b$, $\left(Dist(x,u)\right)^\frac32$ should first decrease and then increase, having exactly $1$ local minimum. Formally, this makes it a convex function, and $f(x)$, as well as $ans_u$, share the same property.

Third, by considering the gradient of $f(v_i)$

we are able to judge whether it is worth searching for a better root $c$ in subtree $v_i$. Here, all nodes $x$ inside subtree $v_i$ needs to be take into our consideration. If the overall decline of moving towards $v_i$, in an unformal way, is even greater than the sum of inclination of all other children, it is possible that node $c\in v_i$ provides a better answer.

Finally, let’s use centroid decomposition. Select a tree’s center, which guarantee that sizes of all it’s subtrees will not exceed half of its own size, will allow us to check the answer of all possible roots within $O(\log N)$. Overall time complexity is $O(N\log N)$.
