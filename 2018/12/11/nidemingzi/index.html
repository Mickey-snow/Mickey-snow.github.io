NOI2018 Nide Mingzi

Posted on 2018-12-11 | Post modified: 2023-08-29 | In Data Structures
Words count in article: 1.8k character(s) | Reading time ≈ 11 minute(s)

We have a string $s$ of length $n\ (1\leq n\leq 5\times 10^5)$ and are presented with $q\ (1\leq q\leq 10^5)$ queries. For each query, given a string $t$ and a segment $[l,r]$, we aim to find the count of unique substrings in $t$ that do not appear in $s_{[l,r]}$. Overall, $\sum |t|\leq 10^6$.

First, construct a suffix tree $T$ for the query string $t$. If $s$ and $t$ have no common characters, the answer is simply the number of leaf nodes on $T$. Next, set a left endpoint $l$ and try to match as much characters as possible, focusing solely on the segment $s_{[l\cdots]}$. If $r$ is the furthest reachable point, mark all nodes that represents $t_{[l,r]}$ in $T$. The final answer is the number of unmarked nodes in $T$.

Second, for queries where $l=1$ and $r=|s|$, align $t$ with the suffix tree $S$. Specifically, assign two pointers to nodes in $S$ and $T$. To shift the starting position to $u+1$, when $t_{[u,v]}$ matches, simply move both pointers to their respective suffix links.

Lastly, in the general case, when considering the segment $s_{[l\cdots r]}$, only visit nodes that lie on a path constructible from suffixes starting between $l$ and $r$. For each node $x$ in the suffix tree, the path from root to $x$ represents perfix for multiple suffixes. Since these suffixes correspond to the leaf nodes in the subtree $x$, we can construct a segment tree for each node $x$ to maintain the existing suffixes in that subtree $x$. By building segment trees of each node and employing persistent segment tree, we can efficiently query for potential suffixes as we traverse to the next node.

The overall time complexity is $O(N\log N+M\log M)$.
